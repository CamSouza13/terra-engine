A — The Inference Loop

How the engine thinks.

It keeps a probability distribution over the loop's true state — a best guess and how sure it is — and every cycle it predicts with a process model, then corrects with whatever sensors reported. The disagreement between the two is what reveals the hidden state no sensor sees.

/01

State model

A mass-balance rulebook for how the loop evolves between readings.

xₖ = f(xₖ₋₁, uₖ) + wₖ
/02

Predict

Roll the rulebook forward through the unscented transform; grow less certain.

x̄, P̄ ← UT(f, x, P) + Q
/03

Correct

Blend in the sensors that reported. Surprise × trust = the nudge.

x ← x̄ + K(z − h(x̄))
/04

Forecast

Sample the posterior, fast-forward, count the futures that breach.

P(breach) = 𝔼[𝟙 g(x_t) > limit]

◇ Two stages, one pipeline. Offline, Hamiltonian Monte Carlo (HMC/NUTS · JAX / NumPyro) fits and validates the process models against real biogeochemical data. Online, the recursive edge estimator shown on this page runs those fitted models live on Raspberry-Pi-class hardware — state and forecast in milliseconds, no network. This page is the edge stage.

◇ Sensors are optional per cycle — drop a probe and the filter leans on the model and the remaining channels. The hidden "health" state (biofilter efficiency, microbial activity, crop capacity) is inferred from the mismatch, never measured directly.

◇ Perception is not action. The engine estimates and forecasts; a separate controller decides the corrective move — water exchange, fertigation, donor dosing, life-support backup — and, only when an operator has authorized autonomy, enacts it. It recommends by default. Switch a loop below to Autonomous to watch it recommend, act, and recover.

01 — Aquaculture RAS.

A recirculating fish farm. Feed becomes ammonia; a biofilter of nitrifying bacteria turns ammonia → nitrite → nitrate. The hidden variable is the biofilter's efficiency — unmeasurable, and the thing that kills stock in hours when it silently fails.
