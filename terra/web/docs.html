API reference

A small, stable HTTP API for enrolling nodes and moving data. All responses are JSON. Base URL is your platform origin.

Authentication

Two credential types, both sent as headers:

Authorization: Bearer sk_live_…      # workspace API key (read endpoints)
X-Node-Key: nk_…                     # per-node key (ingest), with X-Node-Id

Create an API key in the console under Settings → API keys. A node gets its key by redeeming a one-time enrollment token (below). API access requires a paid plan.

Endpoints

POST /api/v1/enroll

Redeem a one-time enrollment token (minted in Settings → Enroll a node) and receive a permanent node key. No auth header — the token is the credential.

curl -s $ORIGIN/api/v1/enroll \
  -H 'Content-Type: application/json' \
  -d '{"enroll_token":"et_…","name":"pond-a","domain":"aquaculture"}'

{ "node_id": "node_ab12cd34", "node_key": "nk_…", "workspace_id": 7 }

The node key is shown once. Store it; the node sends it on every report.

POST /api/v1/ingest

Report a heartbeat: refreshes the node's last-seen time and evaluates alert rules against the supplied metrics.

curl -s $ORIGIN/api/v1/ingest \
  -H 'Content-Type: application/json' \
  -H 'X-Node-Key: nk_…' -H 'X-Node-Id: node_ab12cd34' \
  -d '{"node_id":"node_ab12cd34",
       "domain":"aquaculture",
       "status":{"cycles":128},
       "metrics":{"channels":{"dissolved_oxygen":6.1},"nis":2.4,
                  "risks":{"DO crash":0.08}}}'

{ "ok": true, "node_id": "node_ab12cd34", "workspace_id": 7 }

metrics may carry a channels map, a risks map (name → breach probability 0–1), and scalar nis / hidden. Alert rules match on channel keys, nis, hidden, or risk:<name>.

GET /api/v1/status

Current engine status for the workspace. Requires an API key.

curl -s $ORIGIN/api/v1/status -H 'X-API-Key: sk_live_…'

{ "domain":"aquaculture", "running":true, "cycles":1284, "autonomy":false, … }

GET /api/v1/history?n=200

The last n persisted estimate snapshots (channels, hidden state, NIS). Requires an API key.

curl -s "$ORIGIN/api/v1/history?n=100" -H 'X-API-Key: sk_live_…'

[ { "t": 12.5, "cycles": 50, "channels": {…}, "hidden": {…}, "nis": 1.9 }, … ]

GET /api/v1/config

The provisioning bundle a node applies to run your configured domain and calibrated parameters offline. Authenticate with a node key (or an API key).

curl -s $ORIGIN/api/v1/config -H 'X-Node-Key: nk_…' -H 'X-Node-Id: node_ab12cd34'

{ "domain":"aquaculture", "channels":null, "params":{…}, "safety":[…],
  "speed":0.4, "alerts":[…] }

The node caches this to disk and rebuilds its engine from it, so it runs the same online or offline.

Errors

401 missing/invalid credential · 402 plan doesn't include this · 429 too many attempts. Bodies look like {"error":"…"}.
